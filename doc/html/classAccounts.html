◆ AttemptSignIn()

QString Accounts::AttemptSignIn ( )

AttemptSignIn. verify that username and password exist in text file.

Text file is accessed as read-only and lines are streamed one by one
For each line, password is decrypted using SimpleCrypt, and line info is compared to account's native username and password If there is a match, text file is closed and function returns success. If EOF is reached without a match, function reports that sign in failed.

Returns

◆ AttemptSignUp()

QString Accounts::AttemptSignUp ( )

AttemptSignUp. Reads all needed credentials verifies if they are sound, stores info in text file, and returns result.

Returns
String result of the transaction.

Function will directly terminate without accessing backend text file if:
Any of credentials is empty or NULL
Email format is not deemed correct according to ValidateEmail()
Password doesn't meet requirements (at least 8 characters, containing 1 digit, 1 uppercase and 1 lowercase letters)
Password and confirm password fields don't match

If the above checks pass, function will query text file, and report negative result if:
username or email are already taken by another registered user
If all these flags are passed, function will store all the entered credentials as a string on one line in text file and report success.
Note: Before writing to the text file, password is encrypted using SimpleCrypt
